# 阶段一：数据库层 — A/B 测试配置表实现

> **版本:** 1.0  
> **日期:** 2026-05-18

## 分析与论证

### 现有数据库分析
`database/init.sql` 已包含 18 张表，覆盖用户、电影、行为、推荐缓存等。需要新增 4 张 A/B 测试相关表，并扩展 1 张现有表。

### 关键设计决策

| 决策项 | 方案选择 | 理由 |
|--------|----------|------|
| 分桶算法 | MD5(user_id) → 取前8位hex → 整数 → mod 100 | 确定性分桶，同一用户始终落入同一桶，避免体验跳变 |
| 自适应分流 | 新增 `user_bucket_override` 表覆盖默认桶号 | 新用户通过 Thompson Sampling 决定策略后，记录覆盖映射，后续请求直接命中 |
| 行为表扩展 | 复用 `users_movies_behaviors`，新增 experiment_id / strategy_id 字段 | 无需新建行为表，埋点层零改动 |
| 结果存储 | `ab_results` 表存储聚合指标而非明细 | 避免重复计算，支持看板快速查询 |

### 新增表清单

1. **ab_experiments** — 实验元信息
2. **ab_strategies** — 实验下各策略配置
3. **ab_results** — 离线分析结果汇总
4. **user_bucket_override** — 用户桶覆盖（自适应分流）
5. 扩展 **users_movies_behaviors** — 加 experiment_id / strategy_id
