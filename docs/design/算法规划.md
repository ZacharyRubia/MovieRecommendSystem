构建一个现代化的工业级推荐系统，通常遵循 **“召回 (Recall) -> 过滤 (Filter) -> 排序 (Rank)”** 的经典漏斗架构。结合你的 MySQL + Qdrant 双库设计，以及后续引入 ALS 隐语义模型和深度学习的规划，系统的完整架构设计如下：

------

### 一、 召回层 (多路召回引擎)

召回层的目标是“快且全”，从海量电影（如数万部）中，快速为用户筛选出几百部潜在感兴趣的候选集。我们采用**多路召回**策略：

#### 1. 传统协同过滤 (CF) - 基础行为召回

- **ICF (基于物品的 CF)**：
  - **逻辑**：根据用户近期打高分的电影，推荐与之经常被其他用户共同评分的电影（“喜欢 A 的人也喜欢 B”）。
  - **实现**：利用 Pandas 的 `.corr(method='pearson')` 向量化极速计算电影间的相似度矩阵。
  - **优势**：电影属性相对稳定，推荐结果解释性强（适合放在“相似电影”模块）。
- **UCF (基于用户的 CF)**：
  - **逻辑**：找到与当前用户口味最相似的一批“邻居”用户，推荐他们看过的电影。
  - **痛点与解决**：原生 Python 双重循环计算千万级对用户极慢，必须借助 Pandas 矩阵运算优化，且对极其稀疏的数据表现不佳。

#### 2. 隐语义模型 (ALS 矩阵分解) - 进阶行为召回

- **核心定位**：**用来全面替代/升级传统的 UCF 和 ICF。**
- **逻辑**：通过将庞大的 `(User, Movie)` 稀疏评分矩阵，分解为“用户隐特征向量”和“电影隐特征向量”。预测评分就是两个向量的点乘。
- **实现优势**：
  - 使用 `implicit` 库结合 `scipy.sparse`，在内存友好的情况下，利用 CPU 多核极速完成百万级数据的训练。
  - 生成的 User 向量和 Item 向量不仅能算评分，还能直接计算出更高质量的用户相似度和物品相似度。

#### 3. 基于内容的向量召回 (Content-Based) - 解决冷启动

- **逻辑**：利用大语言模型（如 SentenceTransformers）将电影的文本特征（标题 + 导演 + 演员 + 标签/题材）转化为高维向量。
- **实现**：存入 **Qdrant** 向量数据库。
- **场景**：
  - **新电影冷启动**：新上的电影没人评分，CF 算不出来，但可以通过 Qdrant 搜到与之题材相似的旧电影，推荐给喜欢旧电影的用户。
  - **新用户冷启动**：让新用户先勾选几个偏好标签（写入 `user_preferred_tag` 表），转换成文本向量去 Qdrant 召回初始电影。

------

### 二、 过滤层 (Filter)

从上述 3-4 路召回引擎拿到几百个候选电影 ID 后，必须进行业务规则过滤：

1. **去重**：合并多路召回的结果。
2. **历史行为过滤**：查询 MySQL 的 `user_movie_behavior`，剔除用户已经看过的、点过“不喜欢 (dislike)”的电影。
3. **黑名单/下架过滤**：剔除已下架的视频资源。

------

### 三、 排序与返回层 (Rank & Return)

候选集被过滤到 100 部左右后，需要为它们进行精确打分，决定展示给用户的最终 Top 20 顺序。

#### 1. 现阶段：规则打分与加权融合 (Hybrid 策略)

- **逻辑**：将多路召回的分数归一化后加权相加。比如 ALS 预测分占 40%，ICF 占 30%，Qdrant 相似度占 30%。
- **微调**：可以结合 MySQL `movie.avg_rating` (电影全局平均分) 和 `release_year` (年份)，给高分经典或最新大片一定的提权。

#### 2. 未来进阶：深度学习排序模型

当你掌握了基础架构后，可以通过深度学习彻底重构召回和排序：

- **双塔模型 (Two-Tower Model)**：主要用于**高效召回**。
  - **结构**：左塔输入“用户侧特征”（用户画像、历史观看序列），右塔输入“物品侧特征”（电影元数据）。分别经过多层神经网络（DNN），各自输出一个 Embedding 向量。
  - **工程结合**：将所有电影的“右塔向量”离线刷入 **Qdrant**。线上用户请求时，实时计算“左塔向量”，然后去 Qdrant 做极其高效的 ANN（近似最近邻）搜索。
- **NCF (Neural Collaborative Filtering)**：主要用于**精准排序**。
  - **结构**：传统矩阵分解是把 User 和 Item 向量做简单的内积（线性乘法）。NCF 则是把这两个向量拼接起来，扔进多层感知机（MLP）中，让神经网络去学习极其复杂的非线性交叉特征，输出最终的点击率/评分预测。

------

### 四、 总结：你的系统流转蓝图

1. **数据流 (ETL)**：结构化数据进 **MySQL**，自然语言/特征拼接进 **Qdrant**。
2. **离线层 (T+1 定时任务)**：每天夜里，Python 脚本读取 MySQL 全量行为，训练 **ALS 隐语义模型** 和 **ICF 模型**，并将计算好的“猜你喜欢”JSON 列表写入 MySQL 缓存表，随后同步到 **Redis**。
3. **在线层 (API 服务)**：
   - 前端请求推荐列表。
   - 后端并行触发多路请求：去 Redis 拿 ALS 和 ICF 的缓存，去 Qdrant 拿向量相似结果。
   - 后端执行过滤操作去重去已看。
   - 后端根据加权规则对结果重排，最后去 MySQL `movie` 表关联封面、标题等详细元数据。
   - 组装 JSON 返回前端。
