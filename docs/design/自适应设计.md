## 1. 背景与目标

**问题背景**：在 MovieLens 25M 等大规模评分数据集上，单一推荐算法存在冷启动、覆盖不全、精度不足等问题。协同过滤算法在用户评分稀疏时预测偏差较大。

**设计目标**：

1. 实现 **多种推荐算法**（User-CF、Item-CF、Slope One/Turbo-CF、SVD），支持横向对比
2. 采用 **自适应混合权重**，根据用户活跃度动态调整各算法贡献
3. 建立 **分层缓存体系**（内存缓存 + MySQL 缓存表 + Qdrant 向量缓存），加速推荐响应
4. 支持 **内容推荐降级**（热门/新片/趋势/Qdrant 向量检索），保证始终有推荐结果

## 3. 算法对比与数学原理

### 3.1 传统 User-CF

**公式:**

```
Pearson(u, v) = Σ (r_ui - r̄_u)(r_vi - r̄_v) / √[Σ (r_ui - r̄_u)² · Σ (r_vi - r̄_v)²]
pred(u, i) = (1/k) · Σ r_vi          (等权平均)
```

**实现代码**: `recommendService.js` 中 `computePearson()` 函数（第 563-589 行）

```javascript
// 计算 Pearson 相关系数
function computePearson(ratingsA, ratingsB) {
  // 取共同评分电影
  let sumA = 0, sumB = 0;
  // ... 计算均值
  let num = 0, denomA = 0, denomB = 0;
  for (const { a, b } of common) {
    const dA = a - meanA, dB = b - meanB;
    num += dA * dB;
    denomA += dA * dA;
    denomB += dB * dB;
  }
  return Math.max(0, num / Math.sqrt(denomA * denomB));
}
```

**说明**：
- 先用 SQL 筛选共同评分 ≥ `MIN_COMMON_ITEMS`（3 部）的候选用户
- 再逐批加载候选用户评分数据，计算 Pearson 相关系数
- 最终取 Top-K 邻居的评分等权平均作为预测值

**差异标注**：代码中**未使用去均值偏差**，而是直接对邻居评分取等权平均，与论文公式有所区别。

### 3.2 改进 User-CF

**公式（3-1）:**

```
pred(u,i) = r̄_u + Σ [sim(u,n) · (r_ni - r̄_n)] / Σ |sim(u,n)|
```

**公式（3-3）稳定性因子:**

```
stability(n) = 1 - 1 / (1 + |I_n|)
最终权重 = sim(u,n) · stability(n)
```

**实现状态**: ❌ **未实现于 `recommendService.js`**  
仅在离线评估脚本 `scripts/evaluation/evaluate_models.py` 中作为 `ImprovedUserCF` 类实现（第 272-361 行）。

### 3.3 传统 Item-CF

**公式:**

```
AdjustedCos(i, j) = Σ u∈U_ij (r_ui - r̄_u)(r_uj - r̄_u) / √[Σ (r_ui - r̄_u)² · Σ (r_uj - r̄_u)²]
pred(u, i) = (1/k) · Σ r_uj          (等权平均)
```

**实现代码**: `recommendService.js` 中 `itemBasedCF()` 函数（第 705-851 行）

```javascript
// Cosine 相似度计算
let dot = 0, normA = 0, normB = 0;
for (const [uid, r] of sourceRatings) {
  normA += r * r;
  if (candidateUserRatings.has(uid)) dot += r * candidateUserRatings.get(uid);
}
for (const [, r] of candidateUserRatings) normB += r * r;
const sim = dot / (Math.sqrt(normA) * Math.sqrt(normB));
```

**说明**：
- 通过 SQL JOIN 找到与用户已评分电影有共同评分者的候选电影
- 使用 Cosine 相似度（**未去均值**，即标准的 Cosine 而非 Adjusted Cosine）
- 聚合时使用加权平均（公式 3-2 风格），而不是等权平均

**差异标注**：代码中实际使用 Cosine 相似度而非 Adjusted Cosine，且聚合方式为加权平均。

### 3.4 改进 Item-CF（加权平均）

**公式（3-2）:**

```
pred(u,i) = Σ [sim(i,j) · r_uj] / Σ |sim(i,j)|
```

**实现状态**: ✅ **实现于 `recommendService.js` 中**，`itemBasedCF()` 的聚合部分直接使用加权平均（`weightedSum / simSum`）。  
差异在于相似度计算使用标准 Cosine 而非 Adjusted Cosine。

### 3.5 传统 Slope One（论文公式 3-4 → 实际实现为 Turbo-CF）

**论文公式（3-4）:**

```
dev(j,i) = (1 / |U_ji|) · Σ (r_uj - r_ui)
pred(u,i) = (1 / |R_i|) · Σ (r_uj + dev(i,j))
```

**实际实现**: ⚠️ **项目中的 Slope One 概念已被 Turbo-CF 替代**

- 离线训练脚本: `scripts/train/train_turbocf.py`
- 离线批量推荐: `scripts/recommend/train_recommend.py`
- **`recommendService.js` 中未实现在线 Slope One/Turbo-CF 预测**
- Turbo-CF 的支持仅体现在**数据库缓存表**中（`item_similarity_caches` 和 `user_recommendation_caches` 的 `algorithm` 字段包含 `turbo_cf`）

### 3.6 改进 Slope One（论文公式 3-5,3-6 → 实际未实现）

**公式（3-5）局域偏差:**

```
dev_local(i,j) = avg(r_ui - r_uj), 其中 u ∈ U_nb (邻居集合)
```

**公式（3-6）邻域预测:**

```
pred(u,i) = avg(r_uj + dev_local(i,j)), 其中 j ∈ R_nb
```

**实现状态**: ❌ **完全未实现**  
仅在 `scripts/evaluation/evaluate_models.py` 的 `ImprovedSlopeOne` 类中有离线评估实现。

### 3.7 SVD 模型

**原理:** 使用 TruncatedSVD 对评分矩阵降维，通过用户隐向量与物品隐向量的点积预测评分。

**实现代码**:
- 离线训练: `scripts/train/train_svd.py` → 生成 `svd_model.pkl`
- **`recommendService.js` 中未实现在线 SVD 预测**
- 在线推荐通过**读取 `user_recommendation_caches` 表中 `algorithm = 'svd'` 的缓存**获取结果

### 3.8 混合推荐（公式 3-7）

**公式（3-7）:**

```
score(u,i) = Σ w_k · norm_k(u,i),  其中 Σ w_k = 1
```

**实现代码**: `recommendService.js` 中 `hybridRecommendation()` 函数（第 857-884 行）

```javascript
async function hybridRecommendation(userId, k = DEFAULT_K, topN = DEFAULT_TOP_N, userWeight = null) {
  // 如果 userWeight 未指定，使用自适应权重
  if (userWeight === null) {
    userWeight = await getAdaptiveWeight(userId);
  }
  const itemWeight = 1 - userWeight;
  
  const [userResults, itemResults] = await Promise.all([
    userBasedCF(userId, k, topN * 2),
    itemBasedCF(userId, k, topN * 2)
  ]);
  
  // 加权合并: w_user * user_score + w_item * item_score
  const scoreMap = new Map();
  for (const r of userResults) {
    scoreMap.set(r.movieId, r.predictedRating * userWeight);
  }
  for (const r of itemResults) {
    scoreMap.set(r.movieId, (scoreMap.get(r.movieId) || 0) + r.predictedRating * itemWeight);
  }
  // 排序取 Top-N
}
```

**说明**：
- 混合推荐仅融合 **User-CF + Item-CF** 两个子模型
- 论文公式（3-7）中提及的 SVD、Slope One 等未纳入在线混合
- 混合前**未进行归一化**（论文要求先归一化到 [0,1]），而是直接对原始预测评分加权
- 两个子查询并行 `Promise.all()` 执行

## 4. 自适应混合权重

### 设计预期

论文提出根据用户行为特征（评分数量、多样性、时效性）动态调整权重：

```python
# 理想权重公式
w_user = f(rating_count, rating_diversity, recency)
w_item = 1 - w_user
w_svd = g(user_coverage)
w_slope = h(data_density)
```

### 实际实现

**代码**: `recommendService.js` 中 `getAdaptiveWeight()` 函数（第 439-453 行）

```javascript
async function getAdaptiveWeight(userId) {
  const stats = await query(`SELECT COUNT(*) AS rating_count, ...
                              WHERE user_id = ? AND behavior_type = 'rate'`, [userId]);
  const count = stats[0]?.rating_count || 0;

  if (count < 10) return 0.3;    // 新用户 → 倾向 Item-CF
  if (count < 50) return 0.5;    // 一般用户 → 均衡
  return 0.7;                     // 活跃用户 → 倾向 User-CF
}
```

| 用户类型 | 评分数量 | User-CF 权重 | Item-CF 权重 | 策略说明         |
|----------|----------|:------------:|:------------:|------------------|
| 新用户   | < 10     | 0.3          | 0.7          | 倾向 Item-CF     |
| 一般用户 | 10-49    | 0.5          | 0.5          | 均衡融合         |
| 活跃用户 | ≥ 50     | 0.7          | 0.3          | 倾向 User-CF     |

### 差异分析

| 维度 | 论文设计 | 实际实现 |
|------|----------|----------|
| 因子数量 | 评分数量 + 多样性 + 时效性 | 仅评分数量 |
| 权重粒度 | [0, 1] 连续值 | 3 档离散值 (0.3 / 0.5 / 0.7) |
| 子模型数 | User-CF + Item-CF + SVD + Slope One | 仅 User-CF + Item-CF |
| 归一化 | 需要先归一化各模型输出 | 直接加权原始评分 |

## 5. 数据库缓存设计

### 新旧表对照

### `user_recommendation_caches`（用户推荐缓存）

```sql
CREATE TABLE IF NOT EXISTS `user_recommendation_caches` (
    `user_id`       BIGINT UNSIGNED NOT NULL,
    `algorithm`     VARCHAR(20) NOT NULL,
    `recommend_movies` JSON NOT NULL,
    `updated_at`    TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`user_id`, `algorithm`),
    FOREIGN KEY (`user_id`) REFERENCES `users`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**算法标识（`algorithm` 字段）:**

| 算法标识 | 含义 | 数据来源 |
|----------|------|----------|
| `svd` | SVD 推荐结果 | `scripts/recommend/train_recommend.py` 写回 |
| `user_cf` | User-CF 推荐结果 | `recommendService.saveCacheUserRecommend()` 写回 |
| `item_cf` | Item-CF 推荐结果 | `recommendService.saveCacheUserRecommend()` 写回 |
| `turbo_cf` | Turbo-CF 推荐结果 | 预留（需扩充离线计算） |
| `hybrid` | 混合推荐结果 | `scripts/recommend/train_recommend.py` 写回 |

**JSON 格式示例:**

```json
[
  {"movie_id": 123, "score": 4.5},
  {"movie_id": 456, "score": 4.2}
]
```

### `item_similarity_caches`（物品相似度缓存）

```sql
CREATE TABLE IF NOT EXISTS `item_similarity_caches` (
    `movie_id`       BIGINT UNSIGNED NOT NULL,
    `algorithm`      VARCHAR(20) NOT NULL,
    `similar_movies` JSON NOT NULL,
    `updated_at`     TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`movie_id`, `algorithm`),
    FOREIGN KEY (`movie_id`) REFERENCES `movies`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**算法标识:**

| 算法标识 | 含义 | 数据来源 |
|----------|------|----------|
| `item_cf` | Item-CF 相似度 | 离线计算写回 |
| `turbo_cf` | Turbo-CF 相似度 | 预留 |
| `content_based` | 基于内容的相似度 | 预留 |

### 缓存读策略

- **实时计算**：`recommendService.js` 中的 `userBasedCF()` 和 `itemBasedCF()` 首先尝试读取 `user_recommendation_caches` / `item_similarity_caches`
- **写回时机**：实时计算完成后，通过异步 `saveCacheUserRecommend()` 写回（不阻塞主流程）
- **过期时间**：60 分钟（`CACHE_TTL_MS = 60 * 60 * 1000`）

### 缓存写回工具

- `scripts/recommend/save_to_cache.py`：独立的 Python 缓存写入工具，支持从 JSON 文件或 stdin 批量写回
- 对应旧表名 `users_recommendations`（第 113 行标题与实际 SQL 语句使用 `users_recommendations`），**建议更新为 `user_recommendation_caches`**

### 调用流程（以混合推荐为例）

```
客户端 Request → recommendController.js
  → hybridRecommendation(userId, k=10, topN=10, userWeight=null)
    → getAdaptiveWeight(userId)           // 获取自适应权重
    → Promise.all([
        userBasedCF(userId, k, topN*2),   // 查询 User-CF 结果
        itemBasedCF(userId, k, topN*2)    // 查询 Item-CF 结果
      ])
      ├── userBasedCF:
      │   → getCachedUserRecommend()        // 尝试读缓存表
      │   → findKNearestUsers()             // 找 K 近邻
      │     → SQL: 候选用户筛选
      │     → computePearson()              // 计算 Pearson 相似度
      │   → 聚合邻居评分 → saveCacheUserRecommend() (异步写回缓存)
      │
      └── itemBasedCF:
          → getItemBasedFromCache()         // 尝试读缓存表
          → SQL: 共同评分用户 + 候选电影
          → Cosine 相似度计算 + 聚合
          → saveCacheUserRecommend() (异步写回缓存)
    → 加权合并 User-CF + Item-CF 结果
    → 排序、去重、取 TopN
  → enrichRecommendations()              // 丰富电影元信息
  → Response
```

### 7.4 文档与实际差异汇总

| # | 文档描述 | 实际实现 | 影响 |
|---|----------|----------|------|
| 1 | Slope One 算法 | 项目使用 Turbo-CF 替代，`recommendService.js` 中无在线实现 | 低：离线评估可用 |
| 2 | 旧表 `users_recommendations` | 实际使用 `user_recommendation_caches` | 低：`init.sql` 已清理旧表 |
| 3 | `save_to_cache.py` 旧表名 | 代码第 113/114 行仍引用 `users_recommendations` | **中：需更新** |
| 4 | 自适应权重连续计算 | 3 档离散阈值（<10 / 10-49 / ≥50） | 低：功能正常 |
| 5 | 4 个子模型混合 | 实际仅 User-CF + Item-CF 混合 | 中：论文范围缩小 |
| 6 | 相似度统一使用 Adjusted Cosine | Item-CF 实际使用标准 Cosine | 低：精度差异可接受 |
| 7 | 混合前各模型归一化 | 未归一化，直接加权原始评分 | 中：权重物理意义偏移 |

# 启动后端服务
