# 6 算法数据处理

## 6.1 数据集介绍

本系统采用 MovieLens 大规模电影评分数据集，从 MySQL 数据库中提取了约 **1000 个活跃用户**和 **1000 部热门电影**作为实验子集。数据抽取策略如下：

- **用户选取**：按照评分数量降序排列，优先选取最活跃的 1000 名用户（`extract_test_subset.py` 中 `pick_active_users` 函数实现）
- **电影选取**：从这些用户评过的电影中，按评分出现频率（热度）选取最热门的 1000 部电影（`pick_movies` 函数实现）

抽取后的测试子集包含以下数据文件（存储于 `scripts/extract_test_subset_test/` 目录）：

## 6.2 数据预处理

### 6.2.1 用户数据处理

**数据抽取阶段**（`scripts/extract_test_subset.py`）：

1. **活跃用户筛选**：从 `users_movies_behaviors` 表中按 `behavior_type = 'rate'` 筛选评分行为，按评分数量降序排序，选取前 1000 名最活跃用户
2. **用户基本信息提取**：从 `users` 表提取 `id, username, email, created_at` 字段
3. **用户偏好标签提取**：从 `users_preferred_tags` 表提取用户偏好的标签关联

**模型训练阶段**（`scripts/recommend/train_recommend.py`）：

1. **用户索引映射**：将原始 `user_id` 映射为 0-based 连续整数索引，便于构建评分矩阵
   ```python
   user2idx = {uid: i for i, uid in enumerate(unique_users)}
   ```
2. **用户评分统计**：计算每个用户的评分均值，用于 SVD 的均值中心化和 User-CF 的去均值加权计算

### 6.2.2 电影数据处理

**数据抽取阶段**：

1. **热门电影筛选**：从用户评分的所有电影中，按评分出现次数降序选取前 1000 部（`pick_movies` 函数基于 `users_movies_behaviors` 表的统计计数）
2. **电影基本信息提取**：从 `movies` 表提取 `id, title, description, release_year, duration, avg_rating` 字段

**模型训练阶段**：

1. **电影索引映射**：将原始 `movie_id` 映射为 0-based 连续整数索引
   ```python
   movie2idx = {mid: i for i, mid in enumerate(unique_movies)}
   ```
2. **电影评分统计**：计算每部电影的评分均值，用于 Item-CF 的相似度计算和预测
3. **电影特征提取**：用于 SVD 分解生成电影特征向量，以及 Item-CF 的相似度矩阵计算

### 6.2.3 评分处理及数据合并

**核心评分处理流程**：

1. **评分数据加载**：从 `test_ratings.csv` 读取约 2200 万条评分记录，包含 `user_id, movie_id, rating, created_at` 四个字段

2. **训练/测试集划分**（`train_test_split` 函数）：
   - **按用户分组划分**：对每个用户的评分记录随机打乱，确保每个用户在训练集中至少保留一条记录
   - **测试集比例**：20%（默认 `test_ratio=0.2`）
   - **随机种子**：`random_state=42` 保证可复现

3. **评分矩阵构建**（`build_rating_matrix` 函数）：
   - 构建形状为 `n_users × n_movies` 的稠密评分矩阵
   - 未评分位置使用全局评分均值填充
   - 矩阵稀疏度极高（填充率 < 0.1%），实际训练时对 SVD 使用稀疏矩阵（`csr_matrix`）加速

4. **均值中心化**（SVD）：
   - 对评分矩阵按用户去均值，消除用户评分偏差
   - 每个用户的评分减去其个人评分均值
   - 预测时通过 `pred = dot(user_feat, movie_feat) + user_mean` 还原

## 6.3 模型优化

本系统实现了三种推荐算法，并进行了多项优化：

### 1. SVD 矩阵分解优化

- **Truncated SVD**：使用 `scipy.sparse.linalg.svds` 进行截断奇异值分解，隐因子数设为 50
- **稀疏矩阵加速**：将评分矩阵转换为 `csr_matrix` 稀疏格式，大幅减少内存占用和计算时间
- **均值中心化**：去均值后评分矩阵均值为 0，提升 SVD 分解的数值稳定性
- **奇异值排序**：按奇异值降序排列特征向量，确保保留最重要的潜在特征
- **随机种子设置**：`random.seed(42)` 保证结果可复现

### 2. User-CF 优化

- **Pearson 相关系数**：使用 Pearson 相关系数计算用户相似度，自动消除用户评分偏差
- **向量化相似度计算**：利用 Pandas 的 `pivot()` 和 `corr()` 方法实现高效向量化计算，替代原始的逐对嵌套循环
  ```python
  user_movie_matrix = train_df.pivot(index='user_id', columns='movie_id', values='rating')
  sim_df = user_movie_matrix.T.corr(method='pearson', min_periods=5)
  ```
- **协同评分阈值**：`min_periods=5` — 只有两个用户共同评分过至少 5 部电影才计算相似度
- **邻居数量**：Top-30 最相似邻居（`n_neighbors=30`）
- **仅保留正相关**：过滤掉负相关的用户，避免反向推荐

### 3. Item-CF 优化

- **余弦相似度（去均值）**：使用去均值后的 Cosine 相似度计算电影间相似度
- **共同评分过滤**：只有至少 3 个共同评分用户的电影对才计算相似度
- **邻居数量**：Top-30 最相似电影（`n_neighbors=30`）
- **仅保留正相关**：过滤负相似度电影对

### 4. 混合推荐策略（`recommend.py`）

- **加权融合**：将 SVD、User-CF、Item-CF 的结果进行加权融合，默认权重 `{svd: 0.4, user_cf: 0.3, item_cf: 0.3}`
- **候选池扩展**：各算法取 `top_n × 3` 个候选再进行融合，扩大候选范围
- **自适应容错**：单个算法失败时自动降级，不影响其他算法结果

## 6.4 模型评估与预测结果

### 6.4.1 模型评估方法与指标

**核心评估指标：RMSE（均方根误差）**

系统采用 **RMSE（Root Mean Square Error，均方根误差）** 作为主要评估指标，衡量预测评分与实际评分之间的偏差：

$$RMSE = \sqrt{\frac{1}{n}\sum_{i=1}^{n}(\hat{r}_i - r_i)^2}$$

其中 $\hat{r}_i$ 为预测评分，$r_i$ 为实际评分。

**评估流程**：

1. 将数据集按用户分层划分为训练集（80%）和测试集（20%）
2. 在训练集上训练模型，在测试集上计算 RMSE
3. 分别计算训练集 RMSE 和测试集 RMSE，监控过拟合

**各算法的预测函数**：

- **SVD 预测**：$\hat{r}_{ui} = \text{dot}(U_u, V_i) + \mu_u$（用户特征向量 × 电影特征向量 + 用户均值）
  ```python
  pred = np.dot(user_features[u], movie_features[m]) + user_means[u]
  ```
- **User-CF 预测**：$\hat{r}_{ui} = \mu_u + \frac{\sum_{v \in N_i} \text{sim}(u,v) \cdot (r_{vi} - \mu_v)}{\sum_{v \in N_i} |\text{sim}(u,v)|}$
- **Item-CF 预测**：$\hat{r}_{ui} = \frac{\sum_{j \in N_i} \text{sim}(i,j) \cdot r_{uj}}{\sum_{j \in N_i} |\text{sim}(i,j)|}$

# 使用特定算法
python recommend.py <user_id> --algorithm svd --top_n 20

# 自定义混合权重 (svd, user_cf, item_cf)
python recommend.py <user_id> --algorithm hybrid --weights "0.5,0.25,0.25"

python recommend.py --interactive
